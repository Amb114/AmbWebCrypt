# 安全政策 (Security Policy)

AmbWebCrypt 是一个纯前端工具。你的数据安全和隐私百分之百取决于你的**浏览器环境**和**密码强度**。

**⚠️使用GCM加密时，密码一定设长！（至少）16位字母数字，或6位随机乱序汉字。**

## 1. 支持的版本

我只对最新发布的版本进行安全维护和功能更新。如果你发现 Bug 或者漏洞，请先确保你是在最新版（或者你克隆的最新 HTML）上操作。

| 版本 | 支持状态 |
| 最新版本 | ✅ 支持中 |
| *d5b或更老版本 | ❌ 已停止支持 |

## 2. 核心安全逻辑

1. **零服务端**：数据不经过任何服务器。只要你没被钓鱼，且你的浏览器没被植入后门，你的数据就是绝对私密的。
2. **硬件级加密**：我们用的是 Web Crypto API 原生硬件加速。如果加密不成功，那多半是浏览器策略限制（见 README 甩锅部分）。

## 3. 漏洞报告 (Reporting a Vulnerability)

如果你发现了什么能绕过加密、或是导致本地数据泄露的高级漏洞：

1. **别在 Issue 里直接公开核心细节**（防止被坏人利用）。
2. **密信联系**：建议使用 README 里提供的 **管理员 P-384 公钥**，把漏洞细节加密后发给我。
3. **Issue 预告**：你可以在 GitHub Issues 里发个预告说“发现安全问题，已发密信”，我会尽快肝代码修复。
